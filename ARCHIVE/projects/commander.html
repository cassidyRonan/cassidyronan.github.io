COMMAND THE VIRTUAL WORLD

About Mech Commander VR

Mech Commander VR is a virtual reality realtime strategy game or VR-TS as we referred to it internally. It was developed as part of my third year project in the BSc in Computing in Games Development at IT Sligo alongside Samuel Colville and Kevin O'Carroll, the project was developed as a proof of concept for the viability of a VR-TS where the player can control their units and use various abilities to turn the tide of the battle in Virtual Reality. My primary role in the project was Project Lead and Lead Programmer, however, I also took on the roles of UX/UI Designer, Quality Assurance and Lead Artist. Various agile and project development practices were also carried out during the duration of the project such as Scrum and MoSCoW.

Mech Commander VR allows you to build, control and send Mechs off to fight an evil AI in the form of Real Time Strategy gameplay from your command center. The game utilises SteamVR and the Unity Game Engine to bring an immersive experience to the player and is designed for use with Valve Index VR Controllers.

Features

Detailed below are a selection of the key features which I developed for Mech Commander VR, these features are a combination of gameplay mechanics as well as behind the scene design systems. Included is also a bit of information about my role as lead designer.

Command Center Station


Given the nature of a Real Time Strategy game, it was quickly decided that the player would control their forces from a command outpost looking over the battlefield. Having grown up with movies and shows that had a futuristic hologram aesthetic for any tech interfaces. I decided to bring this into the games as the player's main user interface. I took responsibility for this command center which consisted of a series of connected panels providing the player information and the ability to build and control units.

 

One of the main components of the command center was the map interface which allowed the player to select a point on the map which would then be translated to the physical map, the context for the action carried out on the map was decided by the user holding down the trigger to call up a radial menu in their hand which had a series of actions sushc as "Build Factory", "Build Miner", "Move Unit" and "Attack Enemy".

 

Another component of this system includes an interconnected base system which utilises a design pattern where the main command center script caches any facilities such as factories or miners for quick access to collect resources or activate various methods in other base components.


Character Animation


One of the area's of the project which I took responsibility for was the logic behind the animation selection of the mecha unit's which the player sent into battle. To accomplish this, an animation controller component was added to each unit which was then cached by various scripts on instantiation to reduce the TryGetComponent() methods impact during runtime, the animaton controller then triggered a set animation through the use of an enum and a method built into the animation component.

 


Character Modelling


As Lead Artist on the team, I took on the design and creation of the Mech Character model using Maya LT as well as Adobe Illustrator to create the textures for the model. The original concept for the mechs was to have interchangable hands for different classes of mechs such as lancers, gunners and paladins with each respective class having different attachments. Keeping this concept in mind, the hands were actually modelled seperately so that when the mech was destroyed, the attachments would fall out.

 

After this point, the model was weight painted in a rather unsuual way given the non-organic nature of the character, this involved painting individual parts, deperated by ball joints with the heaviest weight so every vertice moves and does not warp. The model was then exported out to Mixamo for generation of the various animations.
