About Album Creation Studio

Album Creation Studio was developed as part of a college project to utilise some of the many features of Amazon Web Services and develop an API based serverless application. The inspiration for the project came about as a friend of mine had been taking a large amount of polaroids but due to the fact they were polaroids made it difficult for everyone to get a copy of the photo. This led to the idea of creating a digital photo album with some personality that you can send to your friends and family as a PDF.


Technology Stack

  • C#
  • .NET Core 3.1
  • PDFSharp Core
  • GDI+ Library

  • AWS Lambda
  • AWS API Gateway (REST)
  • AWS DynamoDB
  • AWS DAX
  • AWS Code Commit
  • AWS Simple Queue Service
  • AWS Simple Email Service

Services Utilised

Detailed below are a selection of the key features which I developed for Album Creation Studio, these detail the usage of some of the AWS services used in the project as well as a breakdown of the applications functionality.

AWS API Gateway


The application uses AWS API Gateway to create a REST API Endpoint to allow for the client application to send a PUT Request to the service with the user's chosen image files and their email for the album to be sent to after its creation.

 

The purpose of using a REST API means that it is easy to build multiple client interfaces which can send requests to the service and generate an album, these client facing interfaces could be made as a Web Application, Desktop or even a mobile interface.

 

To further improve the experience for users, the API endpoint was linked up to a Lambda @ Edge function meaning that it will fire up a lambda instance close to the user's location to cut down on latency between the user and serverless applications.


AWS Lambda Applications


As shown in the architecture diagram up above, the Album Creation Studio utilises several AWS Lambda Functions to break up the functionality of the program to help reduce costs and improve the speed of album creation. The Lambda applications utilise .NET Core 3.1 for the underlaying code along with the PDFSharp Core and GDI+ Libraries.

 

These functions are configured as Edge functions as aforementioned in the API section, the functions handle the queuing of the album images, applying a border to each images and then compiling all the photos into a album in the PDF Format. As the focus of the project was to utilise AWS Services, the image processing capabilities of the application have been kept simplistic.


AWS Simple Email Service


Once the service has applied the image processing and compiled the images into a PDF document, it then takes the email intially passed to the API from the DYnamoDB database and sends an email containg the album to the end user.

 

After the email has been sent, all user information is removed from the DynamoDB system to ensure a greatly reduced chance of data breaches.
